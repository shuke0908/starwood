# 프로젝트 작업 지침 (Front-end Only)

> [!IMPORTANT]
> 이 프로젝트는 **프론트엔드 완성도**에 모든 역량을 집중합니다. 백엔드, 데이터베이스(Redis 등)와 관련된 실질적인 연동 작업은 수행하지 않습니다.

### 1. 작업 범위 제약

- **UI/UX 최적화**: 디자인 시스템 준수, 부드러운 애니메이션, 모바일 반응형 처리.
- **데이터 관리**: 모든 데이터는 `src/lib/mock-data.ts` 또는 유사한 목데이터 레이어에서 관리합니다.
- **API 레이어**: 실제 API 호출 대신 `Promise`를 반환하는 목업 함수를 사용하여 추후 확장이 용이하도록 구조만 잡습니다.

### 2. 개발 및 배포 환경

- **Docker**: 개발 시 코드 수정이 즉시 반영되도록 `Docker Compose`를 통해 핫 리로드(Hot-reload) 환경을 구축합니다.
- **Vercel**: 프론트엔드 단독 배포를 위한 설정만 유지하며, 서버리스 기능 등 백엔드 의존성을 최소화합니다.

### 3. 일관성 유지

- 모든 컴포넌트는 재사용 가능하도록 설계합니다.
- TypeScript 타입을 엄격히 적용하여 프론트엔드 내부 로직의 무결성을 보장합니다.
